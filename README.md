# Blinkit Sales SQL Analysis

## Project Overview

This project analyzes the Blinkit Grocery Sales dataset using MySQL. The project focuses on data cleaning, exploratory data analysis (EDA), and business insights through SQL queries.

## Dataset

- Dataset: Blinkit Grocery Sales
- Total Records: 7,060
- Database: MySQL

## Project Objectives

- Import CSV data into MySQL
- Clean and validate the dataset
- Perform exploratory data analysis
- Answer business-related questions using SQL
- Practice intermediate and advanced SQL concepts

## SQL Concepts Used

- SELECT
- WHERE
- GROUP BY
- ORDER BY
- HAVING
- Aggregate Functions
- CASE Statements
- Subqueries
- Common Table Expressions (CTEs)
- Window Functions (ROW_NUMBER, DENSE_RANK)

## Analysis Performed

### Data Exploration
- Counted total records
- Identified unique item types
- Previewed the dataset

### Data Cleaning
- Checked for missing values
- Identified duplicate records
- Standardized Item Fat Content values
- Trimmed unwanted spaces
- Validated rating values

### Sales Analysis
- Total, average, minimum, and maximum sales
- Top and bottom performing item categories
- Top-selling products
- Sales percentage by outlet type
- Sales contribution by item fat content

### Outlet Analysis
- Sales by outlet type
- Sales by outlet size
- Sales by outlet location
- Sales by outlet establishment year
- Average ratings by outlet

### Advanced SQL
- Common Table Expressions (CTEs)
- CASE Statements
- Correlated Subqueries
- ROW_NUMBER()
- DENSE_RANK()

## Business Insights

- Identified the highest-performing item categories based on sales.
- Compared outlet performance by type, size, and location.
- Ranked products within each outlet using window functions.
- Categorized products into High, Medium, and Low sales groups.
- Identified products performing above their category average.
